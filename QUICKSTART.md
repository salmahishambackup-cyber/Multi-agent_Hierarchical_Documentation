# Quick Start Guide

Get started with the Multi-Agent Hierarchical Documentation pipeline in 5 minutes.

## Prerequisites

Your project needs these input artifacts (generated by a static analyzer):
```
Artifacts/
└── <your-project>/
    ├── ast.json                          # AST analysis
    ├── dependencies_normalized.json      # Dependency graph
    └── components.json                   # Component groupings
```

## Basic Usage

### Example 1: Generate Module Documentation (Fastest)

```python
from main import run_pipeline

result = run_pipeline(
    repo_root="/content/MyProject",
    artifacts_dir="/content/Artifacts",
    model_id="Qwen/Qwen2.5-Coder-3B-Instruct",
    
    # Recommended settings for T4
    quantize=True,              # 4-bit quantization
    max_input_tokens=2048,      # Truncate long prompts
    docstring_mode="modules_only",  # Only module docs
    
    debug=True,
)

print("Generated files:")
for key, path in result["outputs"].items():
    if path:
        print(f"  {key}: {path}")
```

**Output:**
- `Artifacts/MyProject/doc_artifacts.json` - Generated docstrings
- `Artifacts/MyProject/doc_plan.json` - Processing order
- `Artifacts/MyProject/quality_report.json` - Validation issues
- `MyProject/README.md` - Curated README

### Example 2: Full Documentation (Symbols + Modules)

```python
result = run_pipeline(
    repo_root="/content/MyProject",
    artifacts_dir="/content/Artifacts",
    model_id="Qwen/Qwen2.5-Coder-3B-Instruct",
    quantize=True,
    docstring_mode="symbols_and_modules",  # Full docs
    generate_readme=True,
    readme_use_llm=True,  # Use LLM for README overview
    debug=True,
)
```

### Example 3: No LLM (Artifact Organization Only)

```python
# Fastest: just organize artifacts, no LLM calls
result = run_pipeline(
    repo_root="/content/MyProject",
    artifacts_dir="/content/Artifacts",
    docstring_mode="none",  # Skip LLM
    generate_readme=True,   # Basic README from artifacts
    readme_use_llm=False,
)
```

## Configuration Options

### Memory Optimization (for smaller GPUs)

```python
run_pipeline(
    # ... other params ...
    quantize=True,          # 4-bit quantization (reduces memory ~75%)
    max_input_tokens=1024,  # Reduce from 2048 if still OOM
    keep_loaded=False,      # Unload models after each agent
)
```

### Multiple Projects (Reuse Model)

```python
from Utils.ToolBox.llm_clients.session_router import SessionRouter

# Load model once
router = SessionRouter(
    run_mode="shared",
    shared_model_id="Qwen/Qwen2.5-Coder-3B-Instruct",
    quantize=True,
)

# Process multiple projects
for project in ["/content/Project1", "/content/Project2"]:
    run_pipeline(
        repo_root=project,
        artifacts_dir="/content/Artifacts",
        router=router,  # Reuse model
        docstring_mode="modules_only",
    )

router.cleanup()
```

### Different Models per Agent

```python
run_pipeline(
    # ... other params ...
    run_mode="per_agent",
    agent_models={
        "writer": "Qwen/Qwen2.5-Coder-3B-Instruct",
        "critic": "Qwen/Qwen2.5-Coder-1.5B-Instruct",  # Lighter
        "readme": "Qwen/Qwen2.5-Coder-3B-Instruct",
    },
)
```

## Common Issues

### Out of Memory (OOM)

**Symptoms:** CUDA out of memory error

**Solutions:**
1. Enable quantization: `quantize=True`
2. Reduce input tokens: `max_input_tokens=1024`
3. Use modules_only mode: `docstring_mode="modules_only"`
4. Unload after each agent: `keep_loaded=False`

### Slow Generation

**Symptoms:** Takes too long to complete

**Solutions:**
1. Use smaller model: `"Qwen/Qwen2.5-Coder-1.5B-Instruct"`
2. Use modules_only mode: `docstring_mode="modules_only"`
3. Skip LLM entirely: `docstring_mode="none"`

### Missing Artifacts

**Symptoms:** FileNotFoundError for ast.json

**Solution:** Generate required artifacts using a static analyzer first:
- `ast.json` - AST analysis of your project
- `dependencies_normalized.json` - Dependency graph
- `components.json` - Component groupings

These must exist in `Artifacts/<project_name>/` before running the pipeline.

## Performance Expectations

| Mode | LLM Calls | Time (3B model) | Memory (T4) |
|------|-----------|-----------------|-------------|
| `none` | 0 | < 1 min | < 1 GB |
| `modules_only` | ~10-50 | 5-15 min | 3-4 GB |
| `symbols_and_modules` | ~100-500 | 30-60 min | 3-4 GB |

*Times assume quantization enabled and 10-50 module project*

## Recommended Settings by Use Case

### Quick Documentation (Fastest)
```python
docstring_mode="modules_only"
quantize=True
readme_use_llm=False
```

### Complete Documentation (Best Quality)
```python
docstring_mode="symbols_and_modules"
quantize=True
readme_use_llm=True
llm_params_by_role={
    "writer": {"temperature": 0.2, "max_new_tokens": 300},
    "critic": {"temperature": 0.1, "max_new_tokens": 200},
    "readme": {"temperature": 0.3, "max_new_tokens": 400},
}
```

### Multiple Projects (Most Efficient)
```python
# Use SessionRouter to avoid reloading
router = SessionRouter(run_mode="shared", shared_model_id="...", quantize=True)
# ... process each project with router ...
router.cleanup()
```
